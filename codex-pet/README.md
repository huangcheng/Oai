# Seelie Codex Pet

The Seelie mascot packaged in **OpenAI Codex CLI's pet format**, ready
to upload to https://codex-pets.net.

This is a different bundle from Seelie's *internal* spriteSheet pack at
`assets/packs/seelie/`:

| | Internal Seelie pack | Codex pet (this directory) |
|---|---|---|
| Manifest | `assets/packs/seelie/manifest.json` (Seelie schema) | `pet.json` (Codex CLI schema, 4 fields) |
| Loader | Seelie's `SpritePackManager` + `SpriteAnimationEngine` | Codex CLI's pet picker |
| Atlas | `assets/packs/seelie/spritesheet.webp` | `spritesheet.webp` *(byte-identical to internal pack)* |
| Distribution | Bundled inside `Seelie.app` | Uploaded to codex-pets.net |

Both pets share an identity by design — the same 8×9 atlas powers both,
so the desktop pet and the Codex pet look like the same character.

## Files

- **`pet.json`** — 4-field Codex CLI manifest:
  - `id`: `seelie`
  - `displayName`: `Seelie`
  - `description`: persona summary
  - `spritesheetPath`: `spritesheet.webp` (relative)
- **`spritesheet.webp`** — 1536×1872 WebP, 8×9 atlas, 192×208 cells.
  Generated by OpenAI Codex's `hatch-pet` skill (run 2026-05-17) from
  `artworks/mascot/seelie-avatar_001.jpg` as the row-base reference.

These two files are exactly what codex-pets.net expects — no extra
metadata, no QA artifacts. Upload them as-is, or zip them at this
directory's root:

```
cd codex-pet/seelie
zip ../seelie-codex-pet.zip pet.json spritesheet.webp
```

## Regenerating

To rebuild the atlas (preserving identity across rows), see
`.claude/skills/seelie-codex-pet/SKILL.md`. After a fresh hatch-pet run
produces a new `~/.codex/pets/seelie/`, copy the two files here:

```
cp ~/.codex/pets/seelie/pet.json         codex-pet/seelie/pet.json
cp ~/.codex/pets/seelie/spritesheet.webp codex-pet/seelie/spritesheet.webp
```

And update `assets/packs/seelie/spritesheet.webp` to match if you want
the in-app pet to stay in sync with the Codex pet.
